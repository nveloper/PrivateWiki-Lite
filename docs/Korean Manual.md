# Private Wiki Lite Wiki (Static Version) 운영/이용 메뉴얼

본 문서는 GitHub Pages, Vercel 등 **정적 웹 호스팅(Static Web Hosting)** 을 위해 최적화된 **Private Wiki Lite** 의 운영 및 사용법을 설명합니다.

이 버전은 백엔드 서버(Node.js) 없이 순수 HTML, CSS, JS만으로 구동되며, 보안과 관리의 편리함을 위해 웹 UI에서의 편집 및 업로드 기능이 완전히 제거된 "읽기 전용(Read-Only)" 배포 버전입니다.

---

## 1. 정적 위키 구조 및 문서 작성법

이 위키는 데이터베이스가 없으며, 관리자가 컴퓨터(로컬)에서 직접 마크다운(`.md`) 파일을 작성하여 깃허브(GitHub)에 업로드(Commit/Push)하는 방식으로 운영됩니다.

### 1.1 문서 및 폴더 관리
모든 위키 문서는 `docs/` 폴더 안에서 관리됩니다.
- `docs/` 폴더 안에 원하는 이름으로 새 폴더를 만들거나 `문서이름.md` 형식의 마크다운 파일을 생성하세요.
- 생성된 폴더와 파일 구조가 웹사이트 좌측 사이드바의 **트리 메뉴(목록표)** 에 그대로 반영됩니다.
- 이미지 파일 역시 `docs/` 폴더 내부에 함께 넣어두고 마크다운에서 상대 경로로 불러오거나, `assets/images/` 폴더를 생성하여 관리할 수 있습니다.

### 1.2 검색 및 목록 데이터 갱신 (필수)
새로운 문서를 작성했거나 삭제했다면, 위키가 이를 인식할 수 있도록 목록표(`tree.json`)와 검색 데이터(`search_index.json`)를 갱신해 주어야 합니다.

**[자동 갱신 - GitHub Pages 사용 시]**
- 깃허브에 코드를 푸시(Push)하면 깃허브 액션(GitHub Actions)이 자동으로 빌드 스크립트를 실행하여 목록과 검색 데이터를 갱신합니다. (추가 작업 필요 없음)

**[수동 갱신 - 로컬 테스트나 다른 호스팅 사용 시]**
- 터미널에서 `/` 폴더로 이동한 뒤 아래 명령어를 실행합니다.
  ```bash
  node ghp_build.js
  ```
- 실행 후 `tree.json`과 `search_index.json` 파일이 새로 생성된 것을 확인하고 배포하시면 됩니다.

## 2. 사이트 전체 설정 관리 (`settings.json`)

웹 UI에 있던 글로벌 설정(홈페이지 지정 등)이 모두 하나의 설정 파일로 통합되었습니다.
사이트의 굵직한 설정을 바꾸려면 `/settings.json` 파일을 열어 직접 텍스트를 수정해야 합니다.

```json
{
  "siteTitle": "내 멋진 위키",
  "homeDocument": "folder1/welcome.md",
  "faviconUrl": "assets/favicons/favicon.ico",
  "customFontUrl": "assets/fonts/myfont.woff2",
  "customFont": "NanumGothic"
}
```
- **siteTitle:** 좌측 상단 및 브라우저 탭에 표시될 사이트 이름입니다.
- **homeDocument:** 처음 사이트에 접속했을 때 띄울 메인 문서의 경로입니다. (`docs/` 폴더를 기준으로 작성합니다. 예: `welcome.md`)
- **faviconUrl:** 브라우저 탭에 표시될 아이콘(파비콘) 경로입니다. (웹 주소를 넣거나 `/` 폴더 기준의 경로를 적습니다.)
- **customFontUrl:** 적용할 폰트 파일의 경로입니다.
- **customFont:** 폰트의 이름을 임의로 지정합니다. (영문 추천)

*(참고: 테마 모드나 메인 컬러 팔레트 등 개인 취향에 해당하는 설정은 여전히 브라우저의 우측 상단 톱니바퀴 버튼을 눌러 개별 사용자가 설정할 수 있습니다.)*

## 3. 기본 사용법 및 매크로 기능

정적 버전에서도 문서 열람, 내부 목차(TOC) 스크롤, 검색 기능은 원본과 동일하게 동작합니다. 

특히 마크다운 파일 내에 아래와 같은 매크로 구문을 직접 타이핑해 넣으면 동적 위키와 똑같은 편의 기능을 사용할 수 있습니다.

**1. 리다이렉트 (문서 자동 이동)**
이전 문서의 주소로 접속한 사람을 새 문서로 강제 이동시킬 때 문서 최상단에 적어둡니다.
```markdown
{'{REDIRECT:새폴더명/새로운문서.md}'}
```

**2. D-Day 계산기**
특정 날짜까지 남은 일수나 지난 일수를 자동으로 치환하여 보여줍니다.
```markdown
축제까지 {{DAYS_UNTIL:2026-10-31}}일 남았습니다!
```
